Create A Free Static Website

In this post I will walk you through all the steps that I’ve taken to create and publish this static blog website with Hugo and github pages.

Install Hugo

Depending on your preferred theme, you may have to install a specific version, or the “extended” version of Hugo.

To install Hugo, follow the official installation guide and check the hugo version:

hugo version

Create The Site

Run the following command to generate a static site:

hugo new site blog

Add your preferred theme:

git submodule add \
  https://github.com/jugglerx/hugo-whisper-theme.git \
  themes/hugo-whisper-theme

Run a local server to see what your site looks like:

hugo server

Set Up Repositories

Create two repositories:

  1. a private visualstudio repository that contains the Hugo project and the build pipeline that publishes the static files to the github repository
  2. a public github repository that contains the static website files for hosting: https://github.com/menziess/menziess.github.io

Create your visualstudio repository and push your Hugo project to it:

git remote add origin Menziess@vs-ssh.visualstudio.com:v3/Menziess/blog/blog
git push -u origin --all

Next, modify your config.toml configuration:

baseURL = "https://menziess.github.io"
themesDir = "themes"
languageCode = "en-us"
theme = "hugo-whisper-theme"

Continuous Delivery

Finally we’ll create a build pipeline that builds the static website into the menziess.github.io submodule folder and pushes it to the github repository from where it can be hosted:

trigger:
  - master

pool:
  vmImage: ubuntu-latest

variables:
  user: menziess
  email: stefan_schenk@hotmail.com
  repo: menziess.github.io

steps:
  - script: |
      sudo snap install hugo --channel=extended
    displayName: Install hugo extended

  - script: |
      git submodule add https://github.com/$(user)/$(repo).git public
      rm -rf public/*
    displayName: Add submodule to push generated site to

  - script: hugo
    displayName: Build static site

  - script: |
      cd public
      echo Changed directory to $(pwd)
      git checkout master
      git add --all
      git -c user.name=$(user) -c user.email=$(email) commit -m "commit $(date +"%Y_%m_%d_%I_%M_%p")"
      git push -f https://$(token):x-oauth-basic@github.com/$(user)/$(repo).git master
    displayName: Push generated site to github.io

Generate a github token here, and check the “repo” boxes, then copy-paste the token to a variable in your build pipeline:

Publish Your Site

Create a new post:

hugo new posts/create-a-static-blog-with-hugo.md

Modify the page, and push the changes to trigger a new build.

After about 30 seconds, your website should be published: https://menziess.github.io/

Let me know whether it works for you, or whether this setup can be improved in the comments below!
